    Implement retry transparently for a Client as Middleware.

    Implement retry transparently for a Client as Middleware. For dynamics, this is a bit hard as the server could be busy but it returns 500. Otherwise, other 500 errors due to other request issues may retry a bunch of times until they fail but this should only affect developers. The retry policies herein are based on specific status, specific OS (likely) errors and errors caused potentially by mangled message bodies.

    The new dynamics governer limits are in place and these retry policies take them into account via status TooManyRequests:

    See also
